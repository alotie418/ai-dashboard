# 资产负债表口径确认清单（致会计师）

> 配套文档：`docs/PR7B_BALANCE_SHEET_PLAN.md`（技术侧现状盘点与子 PR 拆分）。
> 本文是面向会计师的**非技术版口径确认清单**，对应该文档 §E 的 K-3/K-4/K-5/K-6/K-9/K-10。
> 状态：待会计师确认。确认前，PR-7B 的 K-3~K-10 相关功能一律不实现（详见每组「④」）。

## 背景说明

我们在开发一款**本地经营记账与管理分析工具**，面向小微企业、个体经营者与跨境卖家。它**不是法定报税/审计系统**，所有输出都标注为「经营管理估算，非法定财务报表」。

目前工具已能登记以下基础数据，但**在编制一张「资产负债表」之前，有一批会计口径问题工具无法自行决定**，需要请您明确。下面按 6 个主题分组，每组说明：① 我们已有的数据 ② 系统目前不能决定的问题 ③ 需要您给出的明确口径 ④ 若不确认，系统将暂不实现的功能。

> 重要前提：本工具**目前没有复式记账（借贷总账）**，数据是「建账期初余额 + 当期实际收付/单据 + 估算」的混合，**现金没有「期初+本期变动=期末」的自动结转**。因此在您确认前，工具只展示「各账目汇总快照」，**不声称这是一张平衡的资产负债表**。

### 现有数据一览

| 数据 | 来源 | 说明 |
|---|---|---|
| 现金/银行账户期初余额 | 用户在「账户」里登记 | 只有建账日余额，无期末滚存 |
| 借款/其他负债期初未偿余额 | 用户在「负债/借款」里登记 | 只有期初，无还款滚存 |
| 固定资产原值 | 用户在「固定资产」里登记 | 只有买入原价，**无折旧/无账面净值** |
| 权益/出资登记金额 | 用户在「权益/资本」里登记 | 增资/撤资/支取等事项流水 |
| 已缴税款 | 用户在「已缴税款」里登记 | 历史缴税记录 |
| 应收账款 | 系统按未收款销售自动汇总 | 真实 |
| 应付账款 | 系统按未付款采购自动汇总 | 真实 |
| 存货 | 系统按在库数量 × 加权平均成本 | 不含税 |
| 本期利润 | 系统按当期收入−成本费用估算 | 单期，无跨期结转 |
| 应交税额（估算） | 系统按用户设的税率估算 | 仅估算，非申报数 |

适用的会计制度档位（用户可切换）：中国 / 美国 / 日本 / 欧盟 / 韩国 / 台湾。**以下每个口径问题，请尽量说明是否各制度通用，还是需要分制度区别对待。**

## K-3｜科目分类与归类（资产 / 负债 / 权益）

**① 我们已有的数据**：上表全部条目（现金、应收、存货、固定资产、应付、借款、应交税额、出资、利润）。

**② 系统目前不能决定的问题**：
- 上述每一项应归入「资产 / 负债 / 所有者权益」的哪一类、对应哪个会计科目。
- 资产/负债是否要再分「流动 / 非流动（一年以内 / 一年以上）」，以及如何判断。
- 是否各会计制度（中/美/日/欧/韩/台）的分类与科目名称需要不同。

**③ 需要会计师给出的明确口径**：
1. 请给出每一类数据对应的标准科目名称与归类（资产/负债/权益）。
2. 是否需要「流动/非流动」二级分类？划分标准是什么（如按到期日是否在一年内）？
3. 个体/独资 与 公司，分类是否不同？
4. 各制度是否需要分别给一套科目表？还是可先用一套通用口径？

**④ 若不确认，系统暂不实现**：不把数据归类成「资产/负债/权益」，不显示分类小计；只保留「各账目汇总快照」。

## K-4｜建账期初余额 与 平衡校验

**① 我们已有的数据**：账户/负债的「期初余额」、固定资产原值、出资登记金额（均为用户在建账时手工录入）。

**② 系统目前不能决定的问题**：
- 建账时，期初余额应如何相互勾稽才算「建账完整」（例如期初资产是否必须等于期初负债+权益）。
- 资产负债表是否必须「资产 = 负债 + 所有者权益」严格相等才允许出表？还是允许显示一个「差额」？
- 在没有复式总账、现金无期末滚存的前提下，期末余额如何确定。

**③ 需要会计师给出的明确口径**：
1. 建账期初的录入规则与最低要求是什么？
2. 是否要求严格平衡（差额必须为 0 才出表）？若允许不平衡，差额应如何标注/解释？
3. 现金、负债的「期末余额」应如何得出（是否需要我们补做收付款的逐笔结转）？

**④ 若不确认，系统暂不实现**：不做「资产=负债+权益」平衡校验、不出正式资产负债表；继续以「未平衡的管理口径快照」呈现。

## K-5｜固定资产折旧

**① 我们已有的数据**：固定资产的「名称、类别、购置日期、原值（买入原价）、状态」。**目前完全没有折旧数据。**

**② 系统目前不能决定的问题**：
- 折旧方法（直线法 / 加速法等）、预计使用年限、残值率应取多少。
- 不同资产类别（如设备、车辆、房屋、电子设备）是否用不同年限/方法。
- 不同会计制度的折旧规则是否不同。
- 资产处置（出售/报废）时的账面处理。

**③ 需要会计师给出的明确口径**：
1. 默认折旧方法是什么？是否允许用户按资产逐项设置？
2. 各资产类别的建议使用年限、残值率分别是多少？
3. 折旧从何时开始计提（购置当月 / 次月 / 按天）？
4. 各制度（中/美/日/欧/韩/台）是否需要分别口径？
5. 资产处置时的账面与损益如何处理？

**④ 若不确认，系统暂不实现**：不计算折旧、不显示固定资产「账面净值」、不产生折旧费用；固定资产只显示「原值」。

## K-6｜应交税费 与 已缴税款

**① 我们已有的数据**：用户登记的「已缴税款」记录；系统按用户设税率得出的「应交税额（估算）」。

**② 系统目前不能决定的问题**：
- 「已缴税款」是否、以及如何与「应交税额（估算）」相抵（得出「尚欠/多缴」余额）。
- 增值税进项是否抵扣、如何抵扣；所得税、附加税如何对冲。
- 应交/已交税费在资产负债表上如何列示（资产侧的预缴 / 负债侧的应交）。

**③ 需要会计师给出的明确口径**：
1. 「已缴税款」与「应交税额估算」是否可对冲？对冲规则是什么？
2. 增值税进项抵扣、各税种对冲是否要纳入本工具，还是明确不做？
3. 税款余额在资产负债表上应列在哪一侧、用什么科目名？
4. 各制度税务处理是否不同？

**④ 若不确认，系统暂不实现**：不做任何税额抵扣/对冲/应交税费计算；「已缴税款」仅作**独立备查汇总**，不参与任何资产/负债/权益合计。

---

## K-9｜利润结转（留存收益 / 未分配利润 / 本年利润）

**① 我们已有的数据**：系统估算的「本期利润」（单期、按收入−成本费用）；用户登记的「权益/出资」事项。

**② 系统目前不能决定的问题**：
- 本期利润如何结转为「未分配利润 / 留存收益」。
- 跨期（上年留存 + 本年利润 − 分红 = 期末留存）的结转规则。
- 是否需要区分「资本公积 / 盈余公积 / 未分配利润」等权益明细。

**③ 需要会计师给出的明确口径**：
1. 本期利润结转未分配利润的规则与时点（按月/按年）。
2. 期初留存收益如何录入？分红/利润分配如何处理？
3. 是否需要「资本公积 / 盈余公积」等权益细分？划分依据是什么？
4. 各制度是否不同？

**④ 若不确认，系统暂不实现**：不做利润结转、不计算「未分配利润 / 留存收益」、不拆分权益明细；权益只显示用户登记的出资事项汇总。

---

## K-10｜多币种折算

**① 我们已有的数据**：各账目可带「币种」（如 CNY / USD），目前**按币种分别展示，不做任何换算**。

**② 系统目前不能决定的问题**：
- 多币种是否需要折算成一种本位币来汇总。
- 采用什么汇率（期末汇率 / 平均汇率 / 历史成本汇率）、折算差额如何处理。

**③ 需要会计师给出的明确口径**：
1. 是否需要折算成本位币？本位币是什么？
2. 各类项目分别用哪种汇率？折算差额计入哪里？
3. 各制度是否不同？

**④ 若不确认，系统暂不实现**：不做任何汇率折算、不做跨币种合计；多币种始终**按币种分别列示**。

## 汇总：在您确认前，工具会保持的状态

- ✅ 已上线：**各账目汇总快照**（各台账余额按币种分别汇总，明确标注「非资产负债表、未做分类与平衡」）。
- ⏸️ 暂不实现（等您确认对应口径后再做）：资产/负债/权益分类（K-3）、平衡校验（K-4）、固定资产折旧与净值（K-5）、税费对冲（K-6）、利润结转与留存收益（K-9）、多币种折算（K-10）。
- 📌 提示：受限于工具当前**没有复式总账、现金无期末自动结转**，即使确认了上述口径，要做出一张「严格平衡的正式资产负债表」可能还需要额外的记账基础建设。若您认为「管理口径的近似资产负债概览（允许列示差额、不声称法定平衡）」即可满足小微经营需求，也请一并告知，我们可据此选择更轻量的实现路径。

> 请尽量在每个问题后标注：是「六制度通用」还是「需分制度区别」；以及该项对小微/个体经营是否**可简化或可不做**。感谢！
