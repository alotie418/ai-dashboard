# WooCommerce 真实店铺 · 人工 QA 准备与验收清单

> 状态：**待执行的人工 QA 清单（只读产出）**。
> 基线：main `e83ca5d` / #339 · schema **v23**。
> 背景：WooCommerce 连接 → 拉单 → 暂存 → 预览 → 提交入账链路的**本地自动化已覆盖**（见 [MVP 状态](./ECOMMERCE_MVP_STATUS.md) §7），但以下项**仍待真实店铺实证**。本清单用于在拿到测试店铺时逐项验收，**不代表这些项已在真实环境验证通过**。
> 执行时建议另建 `ECOMMERCE_WOO_REAL_STORE_QA_RUN.md` 记录实测结果（体例参考 `PR6_REAL_PROVIDER_OCR_QA_RUN.md`）；本清单只列**待验项**，不预填结论。

---

## A. 目标

在一个真实（或官方沙盒）WooCommerce 店铺上，端到端验证 [MVP 状态文档 §8](./ECOMMERCE_MVP_STATUS.md#8-未经真实店铺实证的项统一标注) 列出的 8 项**本地自动化无法替代**的真实环境行为，并跑通「拉单→预览→提交→删 sale→解锁重提」完整闭环。

---

## B. 需要准备的环境

| 项 | 要求 |
|---|---|
| 店铺 | 一个 HTTPS 有效证书的 WooCommerce 站（自建测试站或沙盒）；建议 WooCommerce / WordPress 版本尽量新 |
| HPOS | 若条件允许，准备**开启 HPOS（High-Performance Order Storage）**与**未开启**两种对照站，以观察 payload 差异 |
| 税设置 | 准备**税含价（含税定价）**与**税不含价**两种设置的订单，以验证 `total_mismatch` 守卫 |
| 商品 | 若干测试商品；**故意准备一对同名商品**以制造歧义匹配；部分订单用店内未建的商品名（触发仅描述行） |
| 订单 | 覆盖：`processing` / `pending` / `completed` / `refunded` / `cancelled` / `on-hold` / 含优惠券 / 含 fee / 0 元单 / **超过 50 单**（验证翻页） |
| App 版本 | 打包版或 `npm run electron:dev` |
| 数据安全 | **QA 前用 app 内「导出备份 bundle」备份真实库**；QA 建议在独立 / 空库进行 |

---

## C. 凭证准备（安全约束）

| 项 | 要求 |
|---|---|
| 生成方式 | WooCommerce 后台 → 设置 → 高级 → REST API 生成 Consumer key / secret |
| 权限对照 | 主验 **Read（只读）** 权限 key（重点看只读 key 访问 `system_status` 是否 401/403）；另备 Read/Write 一组对照 |
| **不落文件** | **ck/cs 绝不写入本清单、`_RUN` 记录、commit、截图或任何文件**；截图前遮蔽凭证 |
| QA 后处理 | QA 结束后在 WooCommerce 后台**吊销 / 轮换**本次使用的 key |
| 录入路径 | 仅通过 app 设置页录入（safeStorage 加密入库，渲染进程不接触明文） |

## D. 待验项 —— 真实环境行为（本地自动化无法替代）

> 逐项填「通过 / 不通过 / 观察」；**当前状态：全部「待验证」**。

| # | 待验项 | 期望 | 结果 |
|---|---|---|---|
| D1 | 真实 TLS / 证书链 / HTTP→HTTPS 重定向 | HTTPS 正常连；`http://` 输入被拒（`config`）；重定向站行为符合预期 | ☐ 待验证 |
| D2 | 只读 ck/cs 权限语义 | 只读 key 访问 `system_status` 的真实响应（200 / 401 / 403）与拒因映射一致 | ☐ 待验证 |
| D3 | payload 变异（版本 / HPOS / 税含价 / 插件） | 行金额 / 税 / 运费 / 退款字段解析正确；税含价店触发 `total_mismatch` 保守拒绝而非错误入账 | ☐ 待验证 |
| D4 | `X-WP-TotalPages` 存在性 | 真实站 / CDN / 缓存层是否返回该头；缺失时回退「满页即有下一页」判定正确 | ☐ 待验证 |
| D5 | `modified_after` 服务端过滤 | 传入水位后服务端是否真的降量；即便被忽略，暂存 upsert 幂等仍保证不重复 | ☐ 待验证 |
| D6 | 真实 429 限流 / 退避 | 触发限流时的退避与重试节奏；耗尽后 `throttled` | ☐ 待验证 |
| D7 | 量级性能 | 20 页 × 50 单真实拉取时长、`partial` 续拉体验 | ☐ 待验证 |
| D8 | `store_currency` 设置 UX | 未设币种时的拒绝提示清晰；设置后与真实店铺币种对齐 | ☐ 待验证 |

## E. 待验项 —— 端到端闭环（真实数据）

| # | 步骤 | 期望 | 结果 |
|---|---|---|---|
| E1 | 连接 + 测试连接 | 正确 ck/cs → 显示站点信息；错误 ck/cs → 友好报错**不泄露凭证**；确认无任何「账号+密码」输入 | ☐ 待验证 |
| E2 | 拉单 → 暂存 | 样例订单出现在暂存区；**账本零变化**；同步日志正确；PII 未落库 | ☐ 待验证 |
| E3 | 预览 | 行明细 / 运费 / 税 / 费用 / 退款展示正确；商品名称匹配 / 同名歧义标注 | ☐ 待验证 |
| E4 | 幂等重拉 | 重复拉取不产生重复暂存行 | ☐ 待验证 |
| E5 | 提交入账 | 可提交状态订单成功入账；表头 = Σ 明细；运费不入账；差额分解正确；退款 / 取消 / 部分付款单被对应拒因拒 | ☐ 待验证 |
| E6 | 销售页核对 | 入账订单在销售页可见；金额 / 收款状态 / 溯源正确 | ☐ 待验证 |
| E7 | 删 sale → 解锁重提 | 删除已入账 sale 后暂存单显示「销售记录已删除」+ 解锁按钮；解锁后回 `staged`；重提成功且不重复 | ☐ 待验证 |
| E8 | 换机 / 换用户恢复 | 恢复备份到另一机器后，连接进入「需重新录入凭证」状态（safeStorage 机器绑定） | ☐ 待验证 |

## F. 多语言抽查

在 zh-CN / en 两语言下抽查电商模态的关键文案（拒因码、解锁确认、估算合计标注「估算」），确认无 raw i18n key 直出、占位符正确插值。其余 4 语言（zh-TW / ja / ko / fr）由 `test:locale-ui` 与 i18n 守卫覆盖，人工抽查可选。
