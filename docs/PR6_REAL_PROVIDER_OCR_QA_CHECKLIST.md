# PR-6 真实 Key / OCR 人工验收清单

> 状态：**人工验收清单文档**。本文不改任何业务代码（provider/OCR/Key 存储/IPC/DB/报表/i18n 均不动）。
> 用途：用真实 API Key 逐项跑通 8 家 AI provider + Vision OCR，覆盖功能、错误路径、多语言、会计制度、安全。
> 验收人自带真实 Key；本清单不存储任何 Key。所有「测试连接/对话/工具调用/简报/数据分析/OCR」均通过应用 UI 操作（BYOK）。

---

## A. PR-6 目标

用**真实 Key** 验证 8 家 provider 与 Vision OCR 在桌面应用中的实际可用性——这是代码层无法自动化的最后一道发布闸门。覆盖：连接、对话、工具调用查账、AI 简报、数据分析 AI、错误路径、6 语言 UI、6 会计制度、Vision OCR（PDF/图片/失败/修正/回填）、Key 安全。

> 自动化测试已覆盖：provider registry parity（`check:providers`/`check:agent-providers`）、OCR 字段映射（`check:ocr-mapping`）、错误码→i18n 映射（前端）、provider 名称本地化（`check:provider-names`）。**本清单补的是「真实网络 + 真实 Key」这一段。**

## P. 发布前必须通过项（阻断级）

1. 至少 **3 家主力 provider**（建议 Claude/OpenAI/Gemini 或用户主用的国产家）**C1–C6 全通过**。
2. **测试连接（§E）** 对已配置的每家通过。
3. **错误路径 J1/J2/J9** 通过（空 Key、错 Key、错误不泄露 Key）。
4. **安全 N1/N3/N4** 通过（Key 不入日志/截图、错误不泄露 Key）。
5. **OCR M1/M4/M5** 至少 1 家 OCR-capable 通过（识别→修正→回填，不入账）。
6. **多语言 K1/K2** 至少 zh-CN + en 通过（无裸 key、provider 名本地化）。
7. **额度/限流 J4** 不刷屏、有冷却（避免烧额度）。

> 上述任一不通过 → **不发布**。
